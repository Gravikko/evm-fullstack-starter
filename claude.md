# Правила для компонентов

## Минимальная архитектура
Каждый компонент должен быть **минимальным** и **без лишних файлов**.

### Шаблон структуры:
```
component-name/
├── app/
│   ├── providers.tsx    # Providers (Wagmi, RainbowKit, etc.)
│   ├── layout.tsx       # Layout с providers
│   └── page.tsx         # Главная страница с UI
├── lib/
│   └── config.ts        # Конфигурация (wagmi, chains)
├── hooks/               # ТОЛЬКО если нужны кастомные хуки
│   └── useCustom.ts
├── package.json
├── next.config.ts
├── tsconfig.json
└── README.md
```

### Правила:
- ❌ НЕ создавать лишние файлы/папки
- ❌ НЕ добавлять components/ если можно в page.tsx
- ❌ НЕ создавать utils/helpers если не нужны
- ✅ Переиспользовать файлы из других компонентов (providers, config)
- ✅ Один файл = одна задача
- ✅ Краткий и понятный код

## Переиспользование
- `app/providers.tsx` - копировать из wallet-connect
- `lib/config.ts` - копировать из wallet-connect
- Не дублировать код, переиспользовать оптимальные решения

## Объяснение кода для новичков
- **НЕ писать код автоматически** если юзер новичок
- Сначала **проверить что уже существует** в папке компонента
- **Объяснить только то чего не хватает:**
  - Какие импорты нужны
  - Какая логика должна быть
  - Какие функции/хуки использовать
- Пример: если есть пустой `useTokenBalance.ts` → объяснить что туда писать, но НЕ писать код за юзера
